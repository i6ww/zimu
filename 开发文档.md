# 自动打轴工具 - 开发文档

## 一、项目概述

### 1.1 项目背景
开发一个基于 Web 的字幕自动对齐工具，用户可上传音频和文案，自动生成带时间轴的字幕文件（SRT格式），支持手动调整和预览。

### 1.2 核心功能
- 音频文件上传与播放
- 文案（文本）输入
- 阿里云百练 funASR 自动语音识别与对齐
- 实时预览字幕效果
- 手动调整时间轴（拖拽/输入）
- SRT 字幕文件导出

## 三、阿里云百练 funASR 配置

### 3.1 前提条件

1. **开通服务**：登录[阿里云百炼](https://bailian.console.aliyun.com/)开通 FunASR 服务
2. **获取 API Key**：[获取 API Key](https://help.aliyun.com/zh/model-studio/get-api-key)

### 3.2 模型信息

| 属性 | 说明 |
|------|------|
| **模型名称** | `fun-asr`（当前等同 fun-asr-2025-11-07） |
| **支持语言** | 中文（普通话、粤语、多种方言）、英文、日语 |
| **音频格式** | aac、amr、avi、flac、flv、m4a、mkv、mov、mp3、mp4、mpeg、ogg、opus、wav、webm、wma、wmv |
| **采样率** | 任意 |
| **文件大小** | 单个文件不超过 2GB |
| **音频时长** | 不超过 12 小时 |

### 3.3 费用说明

| 地域 | 单价 | 免费额度 |
|------|------|----------|
| **中国内地** | 0.00022 元/秒 | 36,000 秒（10小时），有效期90天 |
| **国际（新加坡）** | 0.00026 元/秒 | 无 |

### 3.4 存储方案选择

由于 funASR API 需要公网可访问的音频 URL，有两种方案：

#### 方案 A：阿里云 OSS（推荐稳定场景）
- 优点：稳定、有 CDN 加速
- 缺点：有少量存储和流量费用

#### 方案 B：服务器本地存储 + Nginx（推荐轻量服务器）
- 优点：零额外成本，适合个人/小团队使用
- 缺点：服务器带宽有限

**本项目采用方案 B**，具体实现见下方"八、服务器 Nginx 配置"

### 5.2 阿里云百练 SDK 调用

```python
# backend/app/core/funasr.py
import dashscope
from dashscope.audio.asr import Transcription
from http import HTTPStatus
import json
import os
from typing import Optional, List, Dict
import time

class FunASRService:
    """阿里云百练 funASR 服务封装"""
    
    def __init__(self, api_key: str = None):
        dashscope.api_key = api_key or os.getenv("DASHSCOPE_API_KEY")
        dashscope.base_http_api_url = 'https://dashscope.aliyuncs.com/api/v1'
    
    def submit_task(self, file_urls: List[str], language_hints: str = "zh") -> str:
        """提交录音文件识别任务"""
        response = Transcription.async_call(
            model='fun-asr',
            file_urls=file_urls,
            language_hints=[language_hints]
        )
        
        if response.status_code == HTTPStatus.OK:
            return response.output.task_id
        else:
            raise Exception(f"提交任务失败: {response.output.message}")
    
    def wait_for_result(self, task_id: str, poll_interval: float = 2.0) -> Dict:
        """同步等待任务完成并返回结果"""
        while True:
            response = Transcription.fetch(task=task_id)
            
            if response.status_code != HTTPStatus.OK:
                raise Exception(f"查询任务失败: {response.output.message}")
            
            status = response.output.task_status
            
            if status == 'SUCCEEDED':
                results = response.output.results
                for result in results:
                    if result['subtask_status'] == 'SUCCEEDED':
                        return self._fetch_result(result['transcription_url'])
                        
            elif status == 'FAILED':
                raise Exception("ASR 任务执行失败")
            
            time.sleep(poll_interval)
    
    def _fetch_result(self, transcription_url: str) -> Dict:
        """从结果 URL 获取识别结果"""
        from urllib import request
        result = json.loads(request.urlopen(transcription_url).read().decode('utf-8'))
        return result
    
    def parse_subtitles(self, result: Dict) -> List[Dict]:
        """解析识别结果为字幕列表"""
        subtitles = []
        
        for transcript in result.get('transcripts', []):
            for sentence in transcript.get('sentences', []):
                subtitles.append({
                    'begin_time': sentence.get('begin_time', 0) / 1000.0,
                    'end_time': sentence.get('end_time', 0) / 1000.0,
                    'text': sentence.get('text', '')
                })
        
        for i, sub in enumerate(subtitles, 1):
            sub['index'] = i
            
        return subtitles
```

### 5.3 文案对齐算法

用户输入的文案需要与 ASR 识别的文本进行对齐：

```python
# backend/app/core/aligner.py
from difflib import SequenceMatcher
from typing import List, Dict

class TextAligner:
    """文案与识别结果对齐器"""
    
    def align(self, user_texts: List[str], asr_sentences: List[Dict], 
              threshold: float = 0.6) -> List[Dict]:
        """将用户文案与 ASR 识别结果对齐"""
        subtitles = []
        used_indices = set()
        
        for user_text in user_texts:
            user_text_clean = user_text.strip()
            if not user_text_clean:
                continue
            
            best_match = None
            best_score = 0
            best_idx = -1
            
            for i, sentence in enumerate(asr_sentences):
                if i in used_indices:
                    continue
                    
                score = self._similarity(user_text_clean, sentence.get('text', ''))
                
                if score > best_score:
                    best_score = score
                    best_match = sentence
                    best_idx = i
            
            if best_match and best_score >= threshold:
                subtitles.append({
                    'begin_time': best_match['begin_time'],
                    'end_time': best_match['end_time'],
                    'text': user_text_clean
                })
                used_indices.add(best_idx)
            else:
                subtitles.append({
                    'begin_time': 0,
                    'end_time': 0,
                    'text': user_text_clean,
                    'status': 'unmatched'
                })
        
        for i, sub in enumerate(subtitles, 1):
            sub['index'] = i
            
        return subtitles
    
    def _similarity(self, text1: str, text2: str) -> float:
        """计算文本相似度"""
        import re
        t1 = re.sub(r'[^\w\s]', '', text1).lower()
        t2 = re.sub(r'[^\w\s]', '', text2).lower()
        return SequenceMatcher(None, t1, t2).ratio()
```

### 5.4 SRT 文件生成

```python
# backend/app/utils/srt.py
from typing import List, Dict

def generate_srt(subtitles: List[Dict]) -> str:
    """生成 SRT 格式字幕"""
    srt_lines = []
    
    for sub in subtitles:
        index = sub.get('index', 1)
        start = seconds_to_srt_time(sub['begin_time'])
        end = seconds_to_srt_time(sub['end_time'])
        text = sub['text']
        
        srt_lines.append(f"{index}")
        srt_lines.append(f"{start} --> {end}")
        srt_lines.append(f"{text}")
        srt_lines.append("")
    
    return '\n'.join(srt_lines)

def seconds_to_srt_time(seconds: float) -> str:
    """将秒数转换为 SRT 时间格式 (00:00:00,000)"""
    if seconds < 0:
        seconds = 0
    
    hours = int(seconds // 3600)
    minutes = int((seconds % 3600) // 60)
    secs = int(seconds % 60)
    millis = int((seconds % 1) * 1000)
    
    return f"{hours:02d}:{minutes:02d}:{secs:02d},{millis:03d}"
```

### 5.5 服务器本地文件存储

```python
# backend/app/utils/storage.py
import os
import uuid
import shutil
from pathlib import Path
from typing import Optional

class LocalStorage:
    """服务器本地文件存储工具"""
    
    def __init__(self, base_dir: str = "/app/data", base_url: str = None):
        """
        初始化本地存储
        
        Args:
            base_dir: 存储根目录（容器内路径）
            base_url: 访问基础 URL（如 http://your-server-ip:8080）
        """
        self.base_dir = Path(base_dir)
        self.audio_dir = self.base_dir / "audio"
        self.audio_dir.mkdir(parents=True, exist_ok=True)
        
        # Nginx 反向代理地址
        self.base_url = base_url or os.getenv("STORAGE_BASE_URL", "http://localhost:8080")
    
    def save_file(self, file_content: bytes, original_filename: str) -> dict:
        """
        保存上传的文件
        
        Args:
            file_content: 文件字节内容
            original_filename: 原始文件名
            
        Returns:
            {
                "local_path": "/app/data/audio/xxx.mp3",
                "public_url": "http://your-server-ip:8080/audio/xxx.mp3"
            }
        """
        # 获取文件扩展名
        ext = Path(original_filename).suffix.lower()
        if not ext:
            ext = ".mp3"
        
        # 生成唯一文件名
        filename = f"{uuid.uuid4().hex}{ext}"
        file_path = self.audio_dir / filename
        
        # 保存文件
        with open(file_path, 'wb') as f:
            f.write(file_content)
        
        # 生成公网访问 URL
        public_url = f"{self.base_url}/audio/{filename}"
        
        return {
            "local_path": str(file_path),
            "public_url": public_url,
            "filename": filename
        }
    
    def delete_file(self, filename: str):
        """删除文件"""
        file_path = self.audio_dir / filename
        if file_path.exists():
            file_path.unlink()
    
    def cleanup_old_files(self, max_age_hours: int = 24):
        """清理超过指定时间的文件"""
        import time
        current_time = time.time()
        max_age_seconds = max_age_hours * 3600
        
        for file_path in self.audio_dir.iterdir():
            if file_path.is_file():
                file_age = current_time - file_path.stat().st_mtime
                if file_age > max_age_seconds:
                    file_path.unlink()
```

## 八、服务器 Nginx 配置（方案 B）

由于 funASR API 需要公网可访问的音频 URL，我们用 Nginx 在 8080 端口提供静态文件服务。

### 8.1 架构说明

```
用户上传音频
    ↓
后端 (Docker :8000) 保存到 /data/audio/
    ↓
Nginx (Docker :8080) 暴露音频目录
    ↓
生成公网 URL: http://服务器IP:8080/audio/xxx.mp3
    ↓
提交给 funASR API
```

### 8.2 端口规划

| 端口 | 服务 | 说明 |
|------|------|------|
| 80 | Nginx | 前端访问（可选） |
| 3000 | Vue 前端 | Web 界面 |
| 8000 | FastAPI 后端 | API 服务 |
| 8080 | Nginx | 音频文件公网访问 |

### 8.3 防火墙设置

```bash
# 开放必要端口
sudo firewall-cmd --permanent --add-port=3000/tcp   # 前端
sudo firewall-cmd --permanent --add-port=8000/tcp   # 后端 API
sudo firewall-cmd --permanent --add-port=8080/tcp   # 音频文件访问

# 重载防火墙
sudo firewall-cmd --reload

# 或使用 ufw (Ubuntu)
sudo ufw allow 3000/tcp
```

## 九、部署步骤

### 9.1 服务器准备

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo systemctl enable docker
sudo systemctl start docker

# 2. 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 9.2 上传项目

```bash
# 1. 上传项目到服务器
scp -r ./zimu user@your-server:/home/user/

# 2. 进入目录
cd /home/user/zimu

# 3. 创建数据目录
mkdir -p data/audio

# 4. 配置环境变量
cp .env.example .env
vim .env  # 编辑 STORAGE_BASE_URL 为你的服务器地址
```

### 9.3 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d --build

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 访问 http://你的服务器IP:3000
```

### 9.4 常见问题排查

```bash
# 1. 查看容器日志
docker-compose logs backend
docker-compose logs nginx

# 2. 进入容器调试
docker-compose exec backend bash
docker-compose exec nginx sh

# 3. 检查音频文件是否生成
ls -la data/audio/

# 4. 重启服务
docker-compose restart
```

## 十、注意事项

1. **API Key 安全**：务必使用环境变量，不要硬编码在代码中
2. **Nginx 配置**：确保 8080 端口已开放且防火墙允许访问
3. **STORAGE_BASE_URL**：必须填写服务器的实际 IP 地址，否则 funASR 无法访问音频
4. **费用监控**：funASR 按调用时长计费，建议先测试小文件
5. **音频质量**：建议使用清晰的音频以提高识别准确率
6. **文件清理**：生产环境建议配置定时任务清理过期音频文件

## 附录：升级为阿里云 OSS（可选）

如果后期需要更好的稳定性和 CDN 加速，可以升级为阿里云 OSS：

### 1. 开通 OSS 服务

1. 登录 [阿里云 OSS 控制台](https://oss.console.aliyun.com/)
2. 创建 Bucket（选择公共读权限）
3. 获取 AccessKey ID 和 Secret

### 2. 安装依赖

```bash
pip install oss2
```

### 3. 修改存储类

```python
# backend/app/utils/storage.py
import oss2
import uuid

class OSSStorage:
    """阿里云 OSS 存储"""
    
    def __init__(self):
        auth = oss2.Auth(
            os.getenv('OSS_ACCESS_KEY_ID'),
            os.getenv('OSS_ACCESS_KEY_SECRET')
        )
        self.bucket = oss2.Bucket(
            auth,
            os.getenv('OSS_ENDPOINT'),
            os.getenv('OSS_BUCKET_NAME')
        )
    
    def save_file(self, file_content: bytes, original_filename: str) -> dict:
        ext = Path(original_filename).suffix or '.mp3'
        filename = f"audio/{uuid.uuid4().hex}{ext}"
        
        self.bucket.put_object(filename, file_content)
        
        # 返回 OSS 公网 URL
        public_url = f"https://{os.getenv('OSS_BUCKET_NAME')}.{os.getenv('OSS_ENDPOINT')}/{filename}"
        
        return {
            "filename": filename,
            "public_url": public_url
        }
```

### 4. 移除 Nginx 配置

使用 OSS 后，可以移除 docker-compose.yml 中的 nginx 服务和 STORAGE_BASE_URL 环境变量。
